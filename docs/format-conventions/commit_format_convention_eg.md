### Template
```
<type>(<scope>): <description of changes>
```

_(Note: Scope is optional)_

### Common Types

The 5 core commit types for daily development:

- **`feat`**: A new feature.
    
- **`fix`**: A bug fix.
    
- **`refactor`**: Code refactoring (a code change that neither fixes a bug nor adds a feature).
    
- **`docs`**: Documentation updates only (e.g., modifying `README.md`, requirement docs, or comments).
    
- **`test`**: Adding new test cases or modifying existing ones.
    

### Scope _[Optional]_

Used to indicate the specific area, module, page, or service affected by the changes.

_Common scopes: `user`, `order`, `login`, `api`, etc._

### Standard Examples

- **New Feature**:
    
    `feat(user): Implement mobile phone number login function`
    
- **Bug Fix**:
    
    `fix(order): Fix the calculation error of the order amount`
    
- **Refactoring**:
    
    `refactor(api): Refactor the request encapsulation logic`
    
- **Documentation** (Example without scope):
    
    `docs: Update the README.md`
